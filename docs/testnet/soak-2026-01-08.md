```bash

RESTART_AT=60          # restart mid-run to test persistence

for i in $(seq 1 "$N"); do
  TXID=$(rpc sendtoaddress "[\"$DEST\",$AMT]")
  rpc generatetoaddress "[1,\"$MINER_ADDR\"]"

  if [ "$i" -eq "$RESTART_AT" ]; then
    sudo systemctl restart qpb-testnet
    sleep 5
  fi
done
```

## Reorg Drill (Local Devnet)

**Objective:** Verify consensus correctness under network partitions — longest chain should win.

### Test Setup

1. **Partition node2** from the Docker network
2. **Build chain A**: Mine 20 blocks on miner+node1 (height 212)
3. **Build chain B**: Mine 25 blocks on isolated node2 (height 217)
4. **Reconnect node2** and verify all nodes converge to chain B

### Results

| Node | Expected | Actual | Status |
|------|----------|--------|--------|
| miner | Reorg to chain B (217) | Reorged to 217 | ✅ |
| node1 | Reorg to chain B (217) | Stuck at 212, crashed on restart | ❌ |
| node2 | Stay on chain B (217) | Remained at 217 | ✅ |

**Result: ❌ PARTIAL FAILURE — Bug #87 filed**

### Bug Discovered

When node1 (on chain A at height 212) tried to sync from miner (which had reorged to chain B at height 217), the P2P sync failed:

```
INFO qpb_node: node started chain=devnet height=212 tip=6f788cba491ccfbf...
INFO qpb_node: resolved --p2p-connect hostname input=miner:38333 resolved=172.18.0.2:38333
Error: p2p sync failed for all peers
```

**Key Finding**: Live reorg works (miner successfully switched when node2 reconnected), but sync from divergent chain fails.

See **#87** for full details and reproduction steps.

## Known Issues

1. **#86 - Wallet cannot spend unconfirmed change outputs**
   - The single failure was due to fee edge case (`insufficient fee: need 1458 sats, have 1457 sats`)
   - Workaround: Mine a block between each transaction

2. **#87 - P2P sync fails when local chain has diverged from peer**
   - Nodes cannot recover from a reorg via initial sync
   - Live reorgs work; restart-based recovery does not
   - **Impact**: Critical — can cause permanent chain splits

---

## Conclusion

The SHRINCS stateful signature scheme correctly persists signing state across node restarts. This is critical for security — if state were lost, signature indices could be reused, potentially compromising the one-time signature security properties.

The 100% success rate post-restart (60/60) confirms the implementation is production-ready for this aspect.

The reorg drill revealed a **critical P2P sync bug** (#87) that needs to be fixed before mainnet. Live reorgs work correctly, but nodes that miss a reorg cannot recover via restart.
